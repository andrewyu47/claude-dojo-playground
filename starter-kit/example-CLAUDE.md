# Example — a resume-tailoring CLAUDE.md

This is a worked example of a project `CLAUDE.md`, adapted from a real one Andrew uses to tailor his resume. It shows how much a good `CLAUDE.md` can lock in: a fixed workflow, hard rules, and a writing style, so every resume comes out consistent without you re-explaining yourself.

**To use it:** copy this file into your own folder as `CLAUDE.md`, then replace the specifics (projects, links, company names) with your own. It's set up to run against the two files next to it in `starter-kit/`: `resume-example.md` (a master resume) and `job-description-example.md` (a target job). Swap in your real resume and a real posting when you're ready.

---

## When asked to tailor a resume

1. Read the target job description (`starter-kit/job-description-example.md`, or your own).
2. Read the master resume (`starter-kit/resume-example.md`, or your own).
3. Apply ALL rules below.
4. Write the result to `Resume_Tailored.md` in this folder.
5. Render it to `Resume_Tailored.html` using a clean, one-page HTML style.

## Resume rules

1. Unique starting verbs, no verb repeats across the entire resume. Run a verb audit every time.
2. No "philosophy" in the Educator bullet under Thought Leadership.
3. Commas instead of em dashes in bullet points.
4. Every bullet fits on one line. No wrapping. Trim if too long.
5. The resume fits on one page. Tight CSS spacing, concise language.
6. DocOps full title: "DocOps Governance Workbench | Automated Copy Compliance Tool | Python, LangChain".
7. Four core projects, always all four: SAGE, DocOps Governance Workbench, QuickRamp, YouTube Shorts Pipeline.
8. No "AI Governance Framework" entry, it duplicates DocOps.
9. Include "Product Management Specialization (Darden)" where relevant.
10. The Cisco/Splunk manager role is listed as "Splunk", end date "Jan 2026", not "Current".
11. Working links, never placeholder `#` hrefs. LinkedIn: https://www.linkedin.com/in/andrewyu47/ , Portfolio: https://andrewyu.app , LeCun article: https://www.linkedin.com/pulse/lecun-studied-llms-i-cs-lewis-both-learned-limits-language-andrew-yu-kogmc/
12. SAGE subtitle always includes "Hackathon Winner | Figma Make, Pinecone, Veo 3".
13. Summary is at most 2 lines.

## Writing style (no AI patterns)

- No em dashes anywhere. Use commas.
- No "triples", no rule-of-three lists in prose.
- No parallel-structured bullet openings used mechanically.
- Active voice, concrete metrics, plain language.

## Cover letter rules

1. No Workday mentions. Focus on Splunk experience only.
2. Commas instead of em dashes throughout.
